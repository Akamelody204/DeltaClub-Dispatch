# B1-4 · 管理端鉴权与占位接口

> **对应任务**：`后端负责人工作计划.md` · Phase 1 · **B1-4**。  
> **方案（已定）**：与小程序 **分流**——**账号 + 密码** 登录；服务端存 **`bcrypt` 哈希**（`ADMIN_PASSWORD_HASH`）；签发 **与小程序同一 `JWT_SECRET`** 的 JWT，载荷 **`role: admin`**、`openid: ''`（与 **`plan/product/dev/接口表-v0.md` §3**、**§5** 一致）。  
> **非本任务**：指派 / 取消 / 强制状态（**§5.2～5.3**）→ **Phase 2 / B2**；管理端 **Web 前端** 不在本仓库。

---

## 1. 已实现路由

| 方法 | Path | 鉴权 | 说明 |
|------|------|------|------|
| POST | `/api/v1/admin/auth/login` | 无 | `{ username, password }` → **AuthSession** |
| GET | `/api/v1/admin/orders` | Bearer **`admin`** | 分页 + `status?`、`orderId?` 筛选（占位） |
| GET | `/api/v1/admin/orders/:orderId` | Bearer **`admin`** | **OrderDetail**（全单可读） |

**代码**：`server/src/admin/*`；订单查询复用 **`OrdersService.listForAdmin` / `getDetailForAdmin`**。

**玩家订单详情 `GET /api/v1/orders/:id`**：`JwtPayload.role === admin` 时 **可阅任意单**（与 `assertOrderAccess` 扩展一致，便于联调）。

---

## 2. 环境变量

见 **`server/.env.example`** 注释：

| 变量 | 说明 |
|------|------|
| `ADMIN_USERNAME` | 登录名 |
| `ADMIN_PASSWORD_HASH` | `bcrypt` 哈希，**禁止**明文密码进库 |
| `ADMIN_JWT_SUB` | 可选；JWT `sub` 与 `user.id` 默认 `00000000-0000-4000-8000-000000000001` |

**生成哈希**（本地执行一次，将输出写入 `.env`）：

```bash
node -e "console.log(require('bcrypt').hashSync('你的强密码',10))"
```

未配置 `ADMIN_USERNAME` / `ADMIN_PASSWORD_HASH` 时，**POST …/admin/auth/login** 返回 **50000**（提示未配置）。
