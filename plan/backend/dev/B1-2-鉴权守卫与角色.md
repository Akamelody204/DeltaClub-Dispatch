# B1-2 · 鉴权守卫与角色

> **对应任务**：`后端负责人工作计划.md` · Phase 1 · **B1-2**。  
> **目的**：**校验 Bearer JWT**、将 **`JwtPayload` 写入 `req.user`**；**未登录统一 40101**；**角色不符统一 40301**（与 **`plan/product/dev/接口表-v0.md` §7**、`docs/API_CONTRACT.md` 一致）。  
> **依赖**：**B1-1**（JWT 签发、`users.role`）。

---

## 1. 代码位置

| 项 | 路径 |
|------|------|
| `JwtAuthGuard` | `server/src/auth/jwt-auth.guard.ts` |
| `RolesGuard` + `@Roles()` | `server/src/auth/guards/roles.guard.ts`、`server/src/auth/decorators/roles.decorator.ts` |
| `@CurrentUser()` | `server/src/auth/decorators/current-user.decorator.ts` |
| `JwtPayload` | `server/src/auth/jwt-payload.ts` |
| `Express.Request.user` 类型 | `server/src/types/express.d.ts` |
| 受保护路由示例 | `server/src/auth/me.controller.ts`：`GET /me`、`GET /roles-demo/player` |

**全局前缀**：`GET /me` 实际为 **`GET /api/v1/me`**（与 `main.ts` 中 `setGlobalPrefix('api/v1')` 一致）。

---

## 2. 用法（后续 B1-3 / B2 订单接口）

```typescript
@Post('orders')
@UseGuards(JwtAuthGuard, RolesGuard)
@Roles('player')
create(@CurrentUser() user: JwtPayload) {
  // user.sub / user.role / user.openid
}
```

- **顺序**：`JwtAuthGuard` 必须在 **`RolesGuard` 之前**。  
- **仅登录**：`@UseGuards(JwtAuthGuard)`。  
- **管理端 JWT**（B1-4）若与小程序 **分签发**，需另设 `AdminJwtGuard` 或统一载荷内 `role: admin` 后仍用 `RolesGuard`。

---

## 3. 可验证事实

1. **无 Token** 访问 `GET /api/v1/me` → **401**，body `code: 40101`。  
2. **有效 Token** → **200**，`data` 含 `id`、`openid`、`role`。  
3. **`GET /api/v1/roles-demo/player`**：当前用户 **非 player** → **403**，`code: 40301`。
