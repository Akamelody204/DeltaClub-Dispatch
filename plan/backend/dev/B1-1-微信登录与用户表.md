# B1-1 · 微信登录与用户表

> **对应任务**：`后端负责人工作计划.md` · Phase 1 · **B1-1**。  
> **目的**：`wx.login` 的 `code` → 微信 `jscode2session` → **用户落库（openid）** → 签发 **JWT**；与 **`plan/product/dev/接口表-v0.md` §3**、`docs/API_CONTRACT.md` **v0.5** 一致。  
> **联调对齐**：小程序 **M1-1**（登录页换 token）；产品 **P1-1**（接口表 v0.4→v0.5 回填）。

---

## 1. 已实现（代码位置）

| 项 | 说明 |
|------|------|
| `POST /api/v1/auth/wechat/login` | `server/src/auth/auth.controller.ts`；请求体 `{ "code": string }` |
| `POST /api/v1/auth/logout` | 同上；`Authorization: Bearer <JWT>` |
| 用户表 `users` | `server/src/user/user.entity.ts`；字段：`openid`（唯一）、`role`（首登默认 `player`）、`nickname`、`avatarUrl` |
| 微信 | `server/src/wechat/wechat.service.ts`；**`WECHAT_USE_MOCK=true`** 时跳过真实微信（仅本地/CI） |
| JWT | `server/src/auth/auth.module.ts`；载荷含 `sub`、`role`、`openid` |
| 全局前缀 | `GET /health` **无** `/api/v1`；业务在 `/api/v1/*`（`server/src/main.ts`） |
| 业务码 | `ApiException` + `HttpExceptionFilter`：校验失败 **40001**；未登录 **40101**（对齐接口表 §7） |

---

## 2. 环境变量（`server/.env.example`）

| 变量 | 必填 | 说明 |
|------|------|------|
| `DB_*` | 联调是 | MySQL 8；本地可用 `server/docker-compose.yml` |
| `JWT_SECRET` | 生产必 | 开发默认占位，**切勿用于生产** |
| `WECHAT_MINI_APPID` / `WECHAT_MINI_SECRET` | 真机是 | 与小程序 AppID 一致；无凭证时设 **`WECHAT_USE_MOCK=true`** |
| `JWT_EXPIRES_IN` | 否 | 秒，默认 `604800`（7 天） |
