# B1-5 · staging 部署与小程序合法域名

> **对应任务**：`后端负责人工作计划.md` · Phase 1 · **B1-5**（**B1-4 管理端鉴权**按领头人顺序 **后做**）。  
> **目的**：**公网可访问的 HTTPS API（staging）** 跑通 `server/`；配合 **微信公众平台 → 服务器域名 → request 合法域名**；与小程序 **M1-5**、**`Phase0-收口与Phase1开启.md`** Phase 1 目标一致。  
> **与设计端文档的关系**：`plan/design/与其他端的对接说明-非design目录曾由会话改动清单.md` 记载曾改 **`后端负责人工作计划.md`**（仅增加 **Phase 0 已完结** 与 `Phase0-收口与Phase1开启.md` 链接），**不替代**本文部署职责；管理端 IA（**`plan/design/dev/D0-3`**：一期无「指派接单」主按钮）与 **B1-4** 相关，与 **B1-5** 并行知晓即可。

## 0. 彻底闭环：谁牵头 · 后端侧必做

**牵头**：环境从 0 到可联调，**后端负责人**推进部署与 URL 交付；**无服务器 / 无备案域名**时找 **领头人或老板** 开云资源与域名，不阻塞排期空谈。

**后端侧清单（缺一即无法与小程序闭环）**：

| 步骤 | 内容 |
|------|------|
| **域名与解析** | 已备案子域（如 `api-staging.xxx.com`）**A 记录**指向 staging 机器 **公网 IP**（或 LB）。 |
| **机上栈** | **Node** 跑 `server`（或 Docker）；前面 **Nginx / Caddy** 终止 **HTTPS**（Let’s Encrypt 或云证书）；反代到本机进程（见 **`server/docs/DEPLOY-STAGING.md`**）。 |
| **外网自证** | 浏览器或 `curl` 能访问 **`https://<根域名>/health`**，返回 **200** + JSON（健康检查 **不带** `/api/v1`，与 **`接口表-v0.md` §1.2** 一致）。 |
| **业务可连** | 同域下 **`/api/v1/auth/wechat/login`** 等（经 HTTPS）可达，用于与小程序联调登录。 |
| **书面交付** | 将 **正式给前端用的 Base URL** 写进仓库文档（**根 URL，无尾斜杠**），并与接口契约一致：见下 **「Base URL 怎么写」**。 |

**Base URL 怎么写（与契约一致）**：

| 概念 | 写法 | 说明 |
|------|------|------|
| **发给小程序 / 写入 `VITE_APP_API_BASE_URL` 的** | `https://api-staging.xxx.com` | **仅根**；**不要**带 `/api/v1`，**不要**尾斜杠。 |
| **业务请求路径** | `Base + /api/v1/...` | 接口表所列 Path 均含 **`/api/v1`** 前缀。 |
| **健康检查** | `Base + /health` | 全局前缀 **排除** `GET /health`，与 `server/src/main.ts` 一致。 |

回填位置建议：**`plan/product/dev/接口表-v0.md` §1.1 staging 行**、`plan/backend/B0-2-运行环境与域名.md`（若已有 staging 表）、**`server/docs/DEPLOY-STAGING.md`** 示例域名替换说明；小程序 **`miniapp/.env.staging`** 占位替换为真值（与 **M0-3** 一致）。

**结论**：后端先交付 **固定 HTTPS 根地址** 并写进文档，小程序才能改 **baseURL**、配 **微信 request 合法域名**、真机验证（**M1-5**）；**两边缺一就闭环不了**。

## 1. 目标与完成标志

| 完成标志 | 说明 |
|----------|------|
| **HTTPS** | 浏览器/curl 访问 `https://<staging-api-根域名>/health` 返回 **200** + JSON |
| **与小程序一致** | 该根 URL（**无路径后缀**）与 `miniapp/.env.staging` 中 **`VITE_APP_API_BASE_URL`** 一致（见 **`plan/miniapp/dev/M0-3-环境切换与baseURL.md`**） |
| **合法域名** | 小程序账号 **开发 → 开发管理 → 开发设置 → 服务器域名** 中 **request 合法域名** 已添加 **仅根域名**（如 `https://api-staging.xxx.com`，**不要**带 `/api/v1`） |
| **联调记录** | 真机或开发者工具 **成功请求** staging（登录或 `GET /health`），**M1-5** 可勾选；本文 **§5 清单** 由执行人填日期/截图路径 |

**说明**：真实云主机、备案、证书、域名 **由负责人采购/备案**；本文提供 **可执行清单与代码侧约定**，不替代运维决策。

## 2. 技术栈与端口（与仓库一致）

| 组件 | 说明 |
|------|------|
| **Node** | ≥ 18；`server` 构建产物 `dist/main.js` |
| **MySQL 8** | 与 `B0-3`、`.env` 中 `DB_*` 一致；staging 库建议独立 **`clubxcx_staging`** |
| **进程** | **pm2** / **systemd** / 容器任选；须 **崩溃自启**、日志可查看 |
| **TLS** | **Nginx / Caddy / 云 LB** 终止 HTTPS，反代到本机 `127.0.0.1:PORT` |
| **Redis** | Phase 1 可无；抢单锁 **Phase 2** 再开 |

## 3. 环境变量（staging）

复制 `server/.env.staging.example` → 服务器上 **`/path/to/app/.env`**（**不入库**）。至少包含：

- `NODE_ENV=staging`
- `PORT`（与反代 upstream 一致）
- `DB_*`、`JWT_SECRET`（**强随机**）、`WECHAT_MINI_APPID`、`WECHAT_MINI_SECRET`（与小程序 **同一 AppID**）
- `DB_SYNC`：staging 建议 **`false`**，改表走迁移脚本（若团队仍用同步，须书面承担风险）

微信 **`request 合法域名`** 要求 **HTTPS**；证书可用 Let’s Encrypt / 云厂商证书。

---

## 4. 微信公众平台配置（request 合法域名）

1. 登录 [微信公众平台](https://mp.weixin.qq.com/) → **开发** → **开发管理** → **开发设置** → **服务器域名**。  
2. **request 合法域名**：填 **API 根** 的 **域名部分**，如 `https://api-staging.example.com`（与接口表 **`plan/product/dev/接口表-v0.md` §1.1**、**`docs/API_CONTRACT.md` §1** 一致后回填）。  
3. 若平台要求 **域名校验文件**：将下载的文件放到 **`https://根域名/校验文件名`** 可访问（可由 Nginx 静态目录或 Nest 静态路由临时提供，**勿**把校验文件名写进业务接口路径）。  
4. **业务域名**、**socket 合法域名** 等按是否使用再开；一期仅 REST 时通常 **只配 request**。

**协同**：配置人可为 **负责人/运营**；后端提供 **准确 URL 列表**（本文 + `B0-2`）。

## 7. 需同步其他端（领头人可代发群公告）
