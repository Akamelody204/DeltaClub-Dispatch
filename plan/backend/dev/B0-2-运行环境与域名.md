## 1. 环境定义

| 环境 | 用途 | 数据 |
|------|------|------|
| **dev** | 开发者本机；可连本地 MySQL/Redis 或 dev 共用库（团队约定） | 可随意造数 |
| **staging** | **对俱乐部演示、小程序联调**；与生产配置方式一致，密钥不同 | 非真实用户数据；可定期重置 |
| **production** | 正式上线 | 真实数据；须备份策略（Phase 3 细化） |

**原则**：配置 **只经环境变量** 注入，代码中不写死域名、库密码、微信 AppSecret。

---

## 2. 域名与 HTTPS（占位表）

以下 **占位符** 在负责人购买域名与证书后替换；本文只锁「结构」。

| 环境 | API Base（示例占位） | HTTPS |
|------|----------------------|-------|
| dev | `http://127.0.0.1:3000` 或本机 hosts | 本地可 HTTP |
| staging | `https://api-staging.example.com` | **必须** HTTPS |
| production | `https://api.example.com` | **必须** HTTPS |

**小程序约束**：staging/production 的 API 域名须在微信公众平台 **开发 → 开发管理 → 开发设置 → 服务器域名** 中配置 **request 合法域名**（仅根域名，子路径不配）。

---

## 3. 环境变量命名（建议）

| 变量 | 说明 |
|------|------|
| `NODE_ENV` | `development` / `staging` / `production` |
| `PORT` | 监听端口，默认如 `3000` |
| `DATABASE_URL` 或分字段 `DB_HOST` `DB_PORT` `DB_USER` `DB_PASSWORD` `DB_NAME` | MySQL 连接 |
| `REDIS_URL` 或 `REDIS_HOST` 等 | Redis（B0-3 与实现一致） |
| `WECHAT_MINI_APPID` `WECHAT_MINI_SECRET` | 微信登录（Phase 1，**不入库**） |

`.env.example` 列出全部键，**值为占位**。
