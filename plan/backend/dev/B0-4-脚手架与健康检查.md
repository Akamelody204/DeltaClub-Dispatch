## 2. 必须提供接口

| 方法 | 路径 | 行为 |
|------|------|------|
| GET | `/health` | **200**，返回 JSON，至少含 `status: "ok"` 与时间戳或版本字段；**不鉴权**。 |

---

## 3. 统一响应与错误（契约骨架）

**成功（示例）**

```json
{
  "code": 0,
  "message": "ok",
  "data": null
}
```

**业务/客户端错误**

- 使用 **稳定业务码**（与后续 `错误码表.md` 对齐）；HTTP 状态建议 **4xx** 表示客户端/鉴权问题，**5xx** 表示服务内部错误。  
- **禁止**向客户端返回堆栈或数据库细节。

**全局过滤器 / 异常管道**： NestJS 使用 `ExceptionFilter`；其它框架等价中间件。未捕获异常映射为 **500 + 统一 body**，并 **服务端打日志**。

### 6.1 可验证事实（审查用）

在仓库 **`server/`** 目录：

```bash
npm install
npm run start
```

另开终端：

```bash
curl -s http://127.0.0.1:3000/health
curl -s http://127.0.0.1:3000/api/v1/meta
```

期望：**HTTP 200**，健康检查 JSON 示例：

```json
{"code":0,"message":"ok","data":{"status":"ok","ts":"…ISO8601…"}}
```

404 示例（统一错误体）：

```bash
curl -s http://127.0.0.1:3000/no-such-route
```

期望：**HTTP 404**，`{"code":404,"message":"…","data":null}`。

**说明**：推荐 **`pnpm`**；当前环境若无 `pnpm`，可用 **`npm`**（已提交 `package-lock.json`）。**B1-1** 起需 MySQL 与 `.env`，见 `server/README.md`。
