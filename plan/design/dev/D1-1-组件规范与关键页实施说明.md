## 2. 色彩（Design Tokens）

| Token | 色值 | 用途 |
|-------|------|------|
| `--color-primary` | `#076BFF` | 主按钮填充、Tab 选中、关键链接 |
| `--color-primary-pressed` | `#0558D6` | 主按钮按下（近似 10% 深） |
| `--color-text-primary` | `#111111` | 标题、正文主色 |
| `--color-text-secondary` | `#8F8F94` | 辅助说明、占位（与微信灰接近） |

## 3. 字号与字重（小程序 rpx）

| 级别 | 建议 | 用途 |
|------|------|------|
| T1 | 40rpx / 600 | 页面大标题（少用） |
| T2 | 34rpx / 600 | 卡片标题、订单号强调 |
| T3 | 30rpx / 500 | 列表主行、按钮内文字 |
| T4 | 28rpx / 400 | 正文、表单标签 |
| T5 | 24rpx / 400 | 辅助信息、时间、状态副文案 |

**行高**：正文 **1.45～1.5**；按钮内 **垂直居中** 即可。

## 4. 组件样式约定

### 4.1 按钮

| 类型 | 视觉 | 使用场景 |
|------|------|----------|
| **主按钮 Primary** | 填充 `--color-primary`，白字，高 88rpx，圆角 **12rpx** | 每屏 **1 个主行动**（提交订单、抢单等） |
| **次按钮 Secondary** | 白底 + 主色描边 1px，主色字 | 次要操作 |
| **文字按钮 Text** | 无框，主色字 T4 | 列表行内「查看详情」 |

**禁用**：主按钮灰底 `#E5E5EA` + 字 `#C7C7CC`；**加载中** 在按钮内显示 loading 或文案「提交中…」（见 **D1-3**）。

### 4.2 列表卡片（订单 / 大厅）

- 白底圆角 **16rpx**，内边距 **24～32rpx**，下间距 **16rpx**。  
- 左侧：状态 **Tag**（小圆角 8rpx，浅底）；右侧或次行：时间 T5。  
- 主信息一行 **T3**，副信息 **T4** `--color-text-secondary`。

### 4.3 表单

- 标签上、控件下；组与组之间 **32rpx** 间距。  
- 输入框：白底 + 底边或全框 `--color-border`，圆角 **8rpx**，高 **80rpx** 起。

### 4.4 底部 Tab（与工程一致）

| 项 | 值 |
|----|-----|
| 未选中字色 | `#7A7E83`（与 `tabBar.color` 一致） |
| 选中字色 | `#076BFF` |
| 背景 | `#FFFFFF` |
| 图标 | 81×81 px 导出；选中和未选中两套 |

## 5. 六页 P0（D0-1）实施要点（默认态）

| 编号 | 页面 | 实施要点 |

| E1 | 陪玩大厅 | 同卡片体系 + 每卡 **抢单** Primary |
| E2 | 陪玩详情 | 状态 + **开始服务 / 完成服务** 分状态展示 Primary |

## 6. 本期「核心参照页」（方案 B 至少 2 页）

以下两页作为 **Spacing / 组件组合** 的优先对齐目标，开发可先按本文 + 两页精调，其余页复用组件：

| 优先级 | 页面 | 理由 |
|--------|------|------|
| **①** | **下单（S2b）** `order/create` | 表单 + 主按钮 + 合规提示，模板性强 |
| **②** | **订单列表（S3）** `order/list` | 卡片 + 空态 + 状态 Tag，复用到 E1 |

若资源允许，第三优先为 **订单详情（S4）** 或 **陪玩大厅（E1）**。
