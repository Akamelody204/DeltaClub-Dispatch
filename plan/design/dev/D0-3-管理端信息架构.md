# D0-3 · 管理后台（PC Web）信息架构草图

> **对应任务**：设计负责人工作计划 · Phase 0 · D0-3（可选）。  
> **目的**：在 **D0-2 §5** 线框之上，单独固定 **导航与页面层级**、**订单列表 / 详情 / 操作区** 的 **IA 分区**（不写像素、不绑组件库），便于 **管理端演示** 与 **后端 / 前端** 对齐路由与权限；**真源业务规则**仍以 **`../../product/dev/P0-3-一期范围与不做清单.md`**、**`../../product/dev/P0-5-主路径页面清单（设计对接）.md`** 为准。  
> **技术栈提示**（非设计定稿）：选型见 **`../../技术栈选型记录.md`** —— 管理端倾向 **Vue3 + Element Plus**，本 IA 按「顶栏 + 内容区」即可落地为 `Layout` + `router-view`。

## 1. 范围与页面地图（P0-5 §4）

| 代号 | 页面 | IA 层级 | 说明 |
|------|------|---------|------|
| A1 | 登录 | **壳外** | 未登录仅可访问；登录后进入 **壳内** |
| A2 | 订单列表 | **壳内 · 一级** | 默认落地页（可配置 redirect） |
| A3 | 订单详情 | **壳内 · 二级** | 自 A2 进入；面包屑返回 A2 |
| A4 | 仪表盘占位 | **壳内 · 可选** | P2，一期可不在导航出现 |

```text
[登录页 A1]
       │ 成功
       ▼
┌────────────────────────────────────── 壳内 Layout ──────────────────────────────────────┐
│  GlobalHeader：产品名 / 当前用户 / 退出                                                  │
├────────────────────────────────────── 主内容区 ────────────────────────────────────────────┤
│  /orders              →  A2 订单列表（筛选 + 表格 + 分页）                                  │
│  /orders/:id          →  A3 订单详情（字段 + 操作区 + 日志）                               │
│  /dashboard（可选）    →  A4 占位                                                         │
└──────────────────────────────────────────────────────────────────────────────────────────┘
```

**路由命名**：实现层可用 `/admin/orders` 等前缀，本文只描述 **层级关系**。

## 2. 全局壳层（壳内 Layout）

| 分区 | 职责 | 内容占位 |
|------|------|----------|
| **GlobalHeader** | 品牌与账号 | 左侧：产品名「三角洲俱乐部 · 管理」；右侧：管理员昵称或账号、**退出登录** |
| **Main（无侧栏 MVP）** | 单栏后台 | 一期 **不强制左侧菜单**；A2 为默认页，A3 为子页。若二期加侧栏，订单仍为「订单管理 → 列表 / 详情」两级 |
| **Main（可选侧栏）** | 扩展 | 仅一项「订单管理」展开为「全部订单」→ 仍指向 A2，避免空导航 |

**信息架构原则**：**少导航、深内容** —— demo 只保证 **订单可查可兜底**，不铺多模块。

## 3. A2 · 订单列表 — 内容区分区

| 区域 ID | 名称 | IA 职责 | 与交互 |
|---------|------|---------|--------|
| **A2-F** | **筛选区（Filter Bar）** | 收窄当前表格数据集 | 条件变更 → **重置页码为 1** → 请求列表 |
| **A2-T** | **表格区（Data Table）** | 高密度扫读 | 行可选中；**行级操作**仅 **查看详情**（进入 A3） |
| **A2-P** | **分页区（Pagination）** | 翻页与每页条数 | 与 A2-F 结果联动 |

### 3.1 筛选区字段（占位 · 与接口表对齐）

| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 订单状态 | 多选或单选 | 与内部状态枚举一致；须含 **待抢单** 等态，便于筛出 **可指派** 订单（**P0-3 §4.1 B**） |
| 创建时间 | 起止日期 | 闭区间或产品约定 |
| 订单号 / 关键词 | 输入框 | 订单号或玩家标识（规则以后端为准） |

### 3.2 表格列（占位）

| 列 | 用途 |
|----|------|
| 订单号 | 主键展示，可固定左列 |
| 状态 | 标签色与小程序侧 **用户可见名** 可映射两套：对内枚举 + 展示列 |
| 玩家 | 昵称 / ID，脱敏规则产品定 |
| 陪玩 | 接单后展示；空为「—」 |
| 创建时间 | 排序默认 **倒序** |
| 操作 | **查看** → A3 |

### 3.3 列表页 **不包含** 的 IA

| 不包含 | 原因 |
|--------|------|
| 行内「指派」「改派」 | **主路径在 A3 详情**（避免误触）；列表行内 **快捷入口** 可选（跳转详情或轻量弹窗，见 **P0-5**）；**已接单改派** 默认不在一期（**P0-3 §4.3**） |
| 行内「强制改状态」 | 属 **高风险操作**，放在 **A3 详情** 内完成，避免列表误触 |

## 4. A3 · 订单详情 — 内容区分区

| 区域 ID | 名称 | IA 职责 |
|---------|------|---------|
| **A3-B** | **面包屑 / 页头** | `订单列表` → `订单详情`；可选显示订单号标题 |
| **A3-D** | **详情字段区（Description List）** | 只读为主：订单全量字段、玩家/陪玩、时间戳、备注等 |
| **A3-O** | **操作区（Operations）** | 见 **§5** 分层 |
| **A3-L** | **审计日志（Audit Log）** | 时间线：操作者、时间、原状态→新状态、说明 |

**滚动**：建议 **A3-O 顶栏固定或吸底**（实现选型），避免长日志页找不到按钮。

## 5. 操作区（A3-O）分层 IA

按 **风险与频率** 分区，避免「一排平等按钮」导致误操作。

| 层级 | 名称 | 包含操作（一期） | 说明 |
|------|------|------------------|------|
| **L0** | **待抢单指派** | **指派陪玩**（选择目标陪玩 + 确认；调用 **行为 D**） | **仅当订单为待抢单**；与 P0-2 §1.1 互斥于抢单；建议 **主按钮** 或醒目区 |
| **L1** | 常规兜底 | **取消订单**（在业务允许状态下） | 可用次要按钮样式 |
| **L2** | 状态修正 | **强制状态修正**（下拉或弹窗：目标状态 + 原因备注） | 仅在 **P2-1 允许迁移** 内；需二次确认 Modal |
| **—** | 默认不含 | **已接单后改派** | 见 **P0-3 §4.3**；默认一期不做独立业务流 |

**权限 IA**：单角色 **管理员** 即可；若二期分角色，再拆「仅读 / 可操作」视图。

## 6. A1 · 登录（壳外）— IA 要点

| 项 | 说明 |
|----|------|
| 唯一出口 | 登录成功 → **redirect** 至 A2 或 `next` 查询参数指定路径 |
| 失败 | 表单内错误提示，不跳转 |
| 忘记密码 | 一期可 **不提供链接**，避免无后端支撑的死链 |
