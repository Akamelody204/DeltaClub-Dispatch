# D1-2 · 微信小程序安全区与原生组件标注

> **对应任务**：设计负责人工作计划 · Phase 1 · **D1-2**。  
> **目的**：固定 **系统导航栏 / 胶囊按钮 / 底部 Tab 与 Home Indicator / 可滚动内容区** 的标注约定，避免高保真或切图与 **uni-app 编译到 mp-weixin** 的实现脱节；实现细节以微信官方文档与工程为准，本文供 **设计与小程序** 对齐「留多少边距、主按钮放哪里」。  
> **依据**：`D0-1`、`D0-2`；工程路由见 **`../../../miniapp/src/pages.json`**（设计侧只读引用，不修改工程）。

---

## 1. 全局：采用「微信原生导航栏」时的默认布局

当前工程 **未** 全局启用自定义导航（`navigationStyle: custom`），设计稿按 **原生顶栏 + 页面内容区 + 原生 tabBar** 绘制。

| 区域 | 说明 | 设计标注要点 |
|------|------|----------------|
| **状态栏 + 导航栏** | 微信统一绘制；标题来自 `pages.json` 各页 `navigationBarTitleText` | 勿在稿面 **重复画** 与系统重复的顶栏背景条，除非该页改为自定义导航（见 §4） |
| **胶囊按钮区** | 右上角菜单与关闭胶囊，**不可覆盖交互热区** | 自定义顶栏内容时，右侧需 **避让胶囊**（微信提供 `wx.getMenuButtonBoundingClientRect`；设计侧标注「右内边距 ≥ 胶囊左缘」） |
| **页面主内容区** | 导航栏与 tabBar **之间** 的可布局区域 | 列表/表单 **从导航栏下缘开始**；勿假设「全屏从 0 开始」除非 custom |
| **底部 tabBar** | 四 Tab：首页 / 下单 / 订单 / 我的 | 主按钮 **不要** 画进 tabBar 条内；页面内 `[Primary]` 放在 **内容区底部** 或 **表单末**，并预留 **底部安全区**（§2） |
| **非 Tab 页** | 如陪玩大厅、登录 | 使用 **原生返回** 或 `navigateTo` 栈；设计稿保留 **左侧返回** 心智，与系统行为一致 |

## 2. 底部安全区（Home Indicator / 全面屏）

| 场景 | 标注 |
|------|------|
| **固定底栏按钮**（如「提交订单」吸底） | 按钮底边与屏幕底之间预留 **`safe-area-inset-bottom`**；设计说明写：**「底栏上缘距屏幕底 ≥ 34pt（iPhone X 系参考）+ 按钮高度」**，实现用 `env(safe-area-inset-bottom)` 或 uni-app 变量 |
| **tabBar 之上** 的最后一屏内容 | 列表最后一项与 tabBar 顶之间 **≥ 16px～24px** 呼吸间距，避免贴 tab |
| **横屏 / 异形屏** | 以 **最小边距 8px + safe area** 为下限，关键信息不贴左右圆角切口 |

**给开发的备注（可复制到 issue）**：吸底容器建议 `padding-bottom: calc(16rpx + env(safe-area-inset-bottom))`；若与组件库冲突，以组件库为准并在 **D1-1 规范页** 写例外。

## 3. 顶部与滚动

| 项 | 说明 |
|----|------|
| **下拉刷新** | `pages.json` 已对 **订单列表、陪玩大厅** 等开启 `enablePullDownRefresh` 时，稿面 **不需** 单独画「刷新头」；避免与系统下拉动画重叠的固定顶栏 |
| **长列表** | 可滚动区域在 **导航栏下、tabBar 上**；大标题若需沉浸，用 **页面内首屏模块** 模拟，不延伸进系统状态栏（除非 custom nav） |

---

## 4. 何时使用「自定义导航栏」

| 条件 | 建议 |
|------|------|
| 需要 **通顶头图**、**大标题与返回** 完全自控 | 单页设 `navigationStyle: custom`，稿面画出 **状态栏高度 + 导航条高度**（通常 **44px + statusBar**），并标注 **胶囊避让** |
| MVP / 一期 | **默认不推 custom**，减少联调成本；若产品坚持某页通顶，在 **该页单独** 开画板并附本文 §4 |

**设计交付**：若某页使用 custom，须在画板 **单独一帧** 标明「本页 custom navigation」，避免与默认顶栏页混排。

---

## 5. 与 uni-app / 微信小程序组件的关系

| 元素 | 设计侧理解 |
|------|------------|
| **`<button open-type>`** | 微信登录按钮有 **原生样式**；定制时保留 **最小可点区域 44×44pt** |
| **`input` / `textarea`** | 键盘弹起时 **底部按钮** 可能被顶起；稿面可注明「键盘避让由实现处理」 |
| **Toast / Modal** | 使用微信原生或统一封装即可；**不在** 稿面逐帧画 toast，在 **D2-1** 补状态即可 |
