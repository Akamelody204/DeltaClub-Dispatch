# 产品负责人工作计划（对总计划负责）

> **角色**：产品兼运营（见总计划 §2）。  
> **责任边界**：以下列交付物未按期、未达质、或与俱乐部/微信规则冲突导致返工或无法上线，由产品侧牵头补救与对齐；技术实现细节由研发负责，但**需求口径、范围边界、验收标准、审核材料**以本文与总计划为准。  
> **依据文档**：`../三角洲陪玩派单小程序-快速落实计划.md`、`../技术栈选型记录.md`。  
> **执行产出目录**：`plan/product/dev/`（与 `design/dev`、`miniapp/dev` 同层级习惯；**P0 / P1 文档与接口表**见该目录 **`README.md`**）。

---

## 1. 总原则（我承诺遵守）

1. **先冻结再开发**：派单能力（**抢单 + 待抢单指派** 及互斥规则）、一期范围与「不做清单」书面确认前，不私下扩大口径。  
2. **状态机单一真源**：订单状态命名、迁移规则以产品文档为准，变更必须同步后端与小程序并记版本号。  
3. **发版前审核关口**：每次提审前完成微信类目/文案/导流自检表签字（或等价确认）。  
4. **文档只维护一份真源**：接口表、状态机、审核清单以飞书/Notion 约定位置为准（与总计划 §7 一致），避免群聊口头为准。

## 2. 与总计划阶段对齐的任务清单

### Phase 0：对齐与准备（约 3～5 天）

| 序号 | 任务 | 产出 | 完成标志 |
|------|------|------|----------|
| P0-1 | 与俱乐部访谈：完整用户路径、定价如何展示、结算一期是否仅线下 | 流程说明 1 页（可附简易流程图） | 负责人确认已读并回复无异议或已修订 |
| P0-2 | 明确 **派单能力**：**抢单大厅**（陪玩侧）与 **管理端指派**（待抢单态）**一期均交付**，及 **互斥裁决**（见 `P0-2`） | 写入范围文档 | 后端/小程序负责人已知悉 |
| P0-3 | 编写并确认 **一期范围 + 不做清单**（争议/退款/支付/多俱乐部等） | 书面一页 | 全员可见，大需求变更走书面变更 |
| P0-4 | 微信小程序：类目、主体资质、敏感词与导流红线调研 | **审核自检表 v0** | 可勾选执行 |
| P0-5 | 与设计对齐主路径页面清单（4～6 关键页） | 页面列表 + 优先级 | 设计可开工线框/视觉 |

#### Phase 0 白话说明（这几件事分别在解决什么问题）

**Phase 0 整体在干什么**  
开工前约 3～5 天，产品要把「俱乐部到底要什么」「微信让不让上线」「一期做到哪一步」说清楚，并留下**书面记录**。否则后端和小程序会按各自理解开发，后面改需求成本很高。

| 序号 | 一句话理解 | 再展开一点 |
|------|------------|------------|
| **P0-1** | 把业务流程画清楚 | 玩家从打开小程序到下单、陪玩怎么接单、钱一期是不是只线下结——这些要问俱乐部并写成**一页流程**（可很简单），避免大家脑补不一致。 |
| **P0-2** | 一期 **公共抢单 + 客户点名 + 管理指派** | 不点名 → **大厅抢单**；客户 **点名某陪玩** → 仅该人可接；管理员 **指派** 公共单。规则与互斥见 **`dev/P0-2-一期派单模式决策.md` §1 / §1.2**。 |
| **P0-3** | 明确「一期做 / 不做」 | 例如：一期做不做争议、退款、微信支付、多俱乐部等——**写进一页范围文档**，并单列**不做清单**。以后有人加需求，对照这份文档，避免范围偷偷变大。 |
| **P0-4** | 提前排雷微信审核 | 查小程序**类目**、公司**资质**是否够、页面**文案**有没有敏感词或引导私下交易等，整理成一份**可勾选的审核自检表**，发版前按表过一遍。 |
| **P0-5** | 让设计知道先画哪几页 | 和产品总计划一致：主路径上 **4～6 个关键页**（如下单、接单、订单详情、我的等）列成清单并标优先级，设计才知道先出哪几张稿。 |

**「完成标志」列是什么意思**  
不是「你心里觉得好了」，而是**有可核对的外部结果**：例如负责人回复确认、文档全员可见、自检表能一条条勾选——方便别人检查 Phase 0 是否真的做完。

**Phase 0 我的退出标准**：范围冻结（可小改不大改）；P0-2、P0-3、P0-4 已交付；参加全员选型/接口文档位置对齐会。  
**☑ Phase 0 已完结**（2026-04-03）：P0-1～P0-5 与接口表骨架已迁入 **`dev/`**，见 **`dev/README.md`**。全员阶段公告（收口/开启 Phase 1）：**`../Phase0-收口与Phase1开启.md`**。
