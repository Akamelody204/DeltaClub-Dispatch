# P1-2 · 文案包 v0

> **对应任务**：`../产品负责人工作计划.md` · Phase 1 · **P1-2**。  
> **目的**：为小程序与管理端提供 **可落稿** 的按钮、空态、加载、**业务错误 toast**（与 **`接口表-v0.md` §7** 一一映射），避免研发/设计现场编字。  
> **依据**：`P0-1-一期业务流程说明（demo）.md`、`P0-3-一期范围与不做清单.md`（默认无客服外链）、`P0-5-主路径页面清单（设计对接）.md`、`接口表-v0.md` §7。  
> **与 P1-3 关系**：订单状态 **长标签/列表角标** 以本文 **§6** 为 v0；**内部枚举 ↔ 展示名** 冻结以 **`P1-3`** 专表为准，届时仅替换 §6 映射表一行，**不改** toast §5。

---

## 0. 全局约定

| 项 | 约定 |
|----|------|
| **语气** | 简短、中性；不承诺「包赢」；不引导私下交易（见 **P0-4**）。 |
| **字数** | 按钮 ≤8 字为宜；toast ≤20 字为宜（过长截断 + 省略号由端上处理）。 |
| **占位符** | `{name}` 表示动态替换；无则整句降级。 |

---

## 1. 加载与刷新

| 场景 | 文案 |
|------|------|
| 全页加载中 | `加载中…` |
| 按钮内加载 | `请稍候` 或按钮变灰不重复文案 |
| 下拉刷新 | `刷新中`（或仅用系统默认，不重复） |
| 提交中（下单/抢单等） | `提交中…` |

---

## 2. 通用网络与系统（无业务 code 或 code 未知）

| 场景 | 文案 |
|------|------|
| 网络不可用 / 请求失败 | `网络异常，请稍后重试` |
| 服务端 500 / code `50000` | `服务繁忙，请稍后再试` |
| 操作过于频繁 `42901` | `操作太频繁，请稍后再试` |

---

## 3. 登录与权限

| 场景 | 文案 |
|------|------|
| 需要登录（拦截页） | `请先登录` |
| 登录失败（微信授权失败等） | `登录失败，请重试` |
| 未登录 / token 无效 `40101` | `登录已过期，请重新登录` |
| 无权限（角色不符）`40301` | `暂无权限使用该功能` |
| 非订单相关人员 `40302` | `无法操作该订单` |

## 5. 业务错误码 → Toast 文案（与 `接口表-v0.md` §7 对齐）

客户端优先用 **`response.code`** 映射；`message` 可作兜底。

| code | 建议 Toast |
|------|----------------|
| 40001 | `请检查填写内容` |
| 40101 | 见 §3 |
| 40301 | 见 §3 |
| 40302 | 见 §3 |
| 40401 | `订单不存在或已失效` |
| 40901 | `手慢了，订单已被接` |
| 40902 | `订单已由管理员指派` |
| 40903 | `订单状态不允许该操作` |
| 40904 | `该单已指定陪玩，无法改派` |
| 42901 | 见 §2 |
| 50000 | 见 §2 |
