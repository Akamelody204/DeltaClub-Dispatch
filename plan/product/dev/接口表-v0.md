## 1. 全局约定

### 1.1 Base URL

| 环境 | Base（见 **`../../backend/B0-2-运行环境与域名.md`**、**`../../backend/B1-5-staging部署与合法域名.md`**） |
|------|------------------------------------------------------------------|
| dev | `http://127.0.0.1:3000` |
| staging | **HTTPS 根 URL**，与 `miniapp/.env.staging` 中 **`VITE_APP_API_BASE_URL`** 一致；占位示例 `https://api-staging.example.com`，**负责人替换真值** |
| production | 部署后回填 |

### 1.2 路由前缀

| 类型 | 前缀 | 说明 |
|------|------|------|
| 业务 API | **`/api/v1`** | 下文 Path 均省略 Base，且 **含此前缀**（例：`POST /api/v1/orders`） |
| 健康检查 | **`/health`** | **无** `/api/v1` 前缀，与 B0-4 一致 |

### 1.3 统一响应 Envelope（成功）

与 **`../../backend/B0-4-脚手架与健康检查.md`** §3 一致。

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | `number` | **`0`** 表示成功 |
| `message` | `string` | 人类可读 |
| `data` | `object \| array \| null` | 业务载荷 |

### 1.4 统一响应 Envelope（失败）

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | `number` | **稳定业务码**（见 §7），**非** HTTP 状态数字混用（后端实现订单模块时应与全局过滤器对齐，见 §1.5） |
| `message` | `string` | 错误说明；可映射 toast |
| `data` | `null` | 失败时一般为 `null` |

### 1.5 与当前 Nest 过滤器的关系（给后端）

当前 `HttpExceptionFilter` 将 `code` 设为 **HTTP status** 属**临时行为**。落地 **订单模块** 时建议：异常体 `code` 改为 **§7 业务码**，HTTP 状态仍用 401/403/404/409 等；**成功** 一律 `code: 0` + HTTP 200。

### 1.6 鉴权

| 项 | 约定 |
|----|------|
| Header | `Authorization: Bearer <access_token>` |
| 获取 token | §5 `POST /api/v1/auth/wechat/login`（小程序）；管理端见 §6 |
| 角色 | JWT 或 session 内嵌 `role`：`player` \| `companion` \| `admin`；接口行内标注 **所需角色** |

## 3. 认证 / 会话（小程序）

| 方法 | Path | 鉴权 | 请求体 | 响应 `data` |
|------|------|------|--------|-------------|
| POST | `/api/v1/auth/wechat/login` | 无 | **WechatLoginBody** | **AuthSession** |

**WechatLoginBody**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `code` | `string` | 是 | `wx.login` 一次性 `code` |

**AuthSession**

| 字段 | 类型 | 说明 |
|------|------|------|
| `accessToken` | `string` | JWT 或 opaque token |
| `tokenType` | `string` | 固定 `"Bearer"` |
| `expiresIn` | `number` | 秒，可选 |
| `user` | **UserBrief** | 见下 |

**UserBrief**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 用户主键 |
| `openid` | `string` | 微信 openid（脱敏展示规则产品另定） |
| `role` | `string` | `player` \| `companion` \| `admin`（一期可单角色） |
| `nickname` | `string` | 可选 |
| `avatarUrl` | `string` | 可选 |

| 方法 | Path | 鉴权 | 请求体 | 响应 `data` |
|------|------|------|--------|-------------|
| POST | `/api/v1/auth/logout` | Bearer | 空或 `{}` | `null` |

**后端落地（B1-1）**：`POST …/wechat/login`、`POST …/logout` 已在 `server/src/auth` 实现；需 **MySQL**（见 `server/docker-compose.yml`）与 **`JWT_SECRET`**；微信凭证或 **`WECHAT_USE_MOCK=true`** 本地联调。

## 4. 订单（玩家 / 陪玩 · 行为 A～C 等）

### 4.1 玩家创建订单（行为 **A**）

| 方法 | Path | 鉴权 | 说明 |
|------|------|------|------|
| POST | `/api/v1/orders` | Bearer **`player`** | 创建订单；**P0-2 §1.2 点名** |

**请求体 CreateOrderBody**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `skuId` | `string` | 是 | 服务 SKU 标识（一期可枚举 id） |
| `gameRegion` | `string` | 否 | 区服 |
| `durationMinutes` | `number` | 否 | 时长档位 |
| `remark` | `string` | 否 | 备注 |
| `designatedCompanionUserId` | `string \| null` | 否 | **点名陪玩**用户 id；有则 **不进公共大厅** |

**响应 `data`：OrderDetail**（见 §4.4）

### 4.2 玩家订单列表

| 方法 | Path | 鉴权 | Query | 响应 `data` |
|------|------|------|-------|-------------|
| GET | `/api/v1/orders` | Bearer **`player`** | `page`, `pageSize`, `status?` | **PageResult**（`items: OrderListItem[]`） |

**OrderListItem**（摘要字段）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 订单号 |
| `status` | `string` | 内部状态枚举（与 P2-1 一致） |
| `statusText` | `string` | 用户可见文案（**真源**：**`./P1-3-订单状态与用户可见文案映射表-v0.md` §2**） |
| `createdAt` | `string` | ISO8601 |
| `designatedCompanionUserId` | `string \| null` | 是否点名 |

### 4.3 订单详情

| 方法 | Path | 鉴权 | 响应 `data` |
|------|------|------|-------------|
| GET | `/api/v1/orders/:orderId` | Bearer **玩家/陪玩（订单相关）** | **OrderDetail** |

**OrderDetail**（在 ListItem 基础上扩展）

| 字段 | 类型 | 说明 |
|------|------|------|
| `skuSnapshot` | `object` | SKU 快照（品名、展示价等） |
| `playerUserId` | `string` | 玩家 id |
| `companionUserId` | `string \| null` | 接单陪玩 id |
| `remark` | `string \| null` | 备注（B1-3 已返回） |
| `gameRegion` | `string \| null` | 区服（B1-3 已返回） |
| `durationMinutes` | `number \| null` | 时长（B1-3 已返回） |
| `timeline` | `array` | 可选；状态节点（B1-3 为占位数组） |

**后端落地（B1-3）**：`POST/GET /api/v1/orders`、`GET /api/v1/orders/:orderId` 已实现；新建单默认 **`PENDING_GRAB`**；**`GET …/:orderId`** 允许 **玩家本人** 或 **相关陪玩**（已接单 / 待抢单且点名为本人）；状态枚举与 **`docs/STATE_MACHINE.md`** v0.1-draft 对齐。详见 **`plan/backend/B1-3-订单表与CRUD草稿.md`**。

### 4.4 陪玩「可接单」列表（行为 **B**）

| 方法 | Path | 鉴权 | Query | 响应 `data` |
|------|------|------|-------|-------------|
| GET | `/api/v1/companion/orders/available` | Bearer **`companion`** | `page`, `pageSize` | **PageResult**（`items: OrderListItem[]`） |

语义：返回 **公共待抢单** + **点名给当前陪玩** 的单（服务端过滤，**P0-2**）。

### 4.5 陪玩抢单 / 认领（行为 **C**）

失败：见 §7（40901/40902/40301 等）。

### 4.6 陪玩推进状态（开始服务 / 完成等）

|------|------|------|--------|-------------|

**CompanionActionBody**

|------|------|------|------|
| `action` | `string` | 是 | 如 `start_service`、`complete_service`（与 **P2-1** 枚举一致） |

### 4.7 玩家取消订单（仅待分配/规则允许）

## 7. 错误码（业务 `code`）

HTTP 建议与业务码组合实现；客户端以 **`code`** 做分支。

| code | HTTP（建议） | 说明 |
|------|----------------|------|
| 0 | 200 | 成功 |
| 40001 | 400 | 参数校验失败 |
| 40101 | 401 | 未登录或 token 无效/过期 |
| 40301 | 403 | 无权限（角色不符，如非陪玩调用 claim） |
| 40302 | 403 | 无权操作该订单（非当事人） |
| 40401 | 404 | 订单不存在 |
| 40901 | 409 | 订单已被他人抢单 / 已被认领（**P0-2 §1.1**） |
| 40902 | 409 | 订单已被管理员指派 |
| 40903 | 409 | 订单状态不允许此操作 |
| **40904** | 409 | **指派违反点名规则**（已点名未接，管理员改派）（**P0-3 §4.4**） |
| 42901 | 429 | 请求过于频繁 |
| 50000 | 500 | 服务器内部错误 |

**文案**：toast 由小程序按 `code` 映射 **P1-2 文案包**。
