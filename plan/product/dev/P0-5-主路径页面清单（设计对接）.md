# P0-5 · 主路径页面清单（设计对接）

> **对应任务**：产品负责人工作计划 · Phase 0 · P0-5。  
> **目的**：与设计对齐 **4～6 个关键页** 的**优先级、信息结构、空/错状态**，避免主流程返工；**高保真范围**以本清单为准，其余页面可用组件库。  
> **关联**：`P0-1-一期业务流程说明（demo）.md`；`../../三角洲陪玩派单小程序-快速落实计划.md` §2 设计职责；`P0-3-一期范围与不做清单.md`（一期不做项勿设计进主流程）。

---

## 1. 设计范围约定

| 项 | 约定 |
|----|------|
| **高保真（建议 4～6 页） | 下表 **P0** 页面；出主色、按钮、列表卡片、关键信息层级即可。 |
| **中低保真 / 组件库** | **P1** 页面及后台：线框或直接用 Element Plus 等默认样式，产品提供字段清单即可。 |
| **不做进一期 UI** | 支付结果页、评价页、完整客服中心、多租户切换等（见 P0-3）。 |

---

## 2. 小程序 · 玩家端

| 序号 | 页面 | 优先级 | 核心内容与状态 | 备注 |
|------|------|--------|----------------|------|
| S1 | **启动/登录** | P0 | 微信授权登录、加载中、失败重试 | 可与首页合并为同一套视觉 |
| S2 | **首页 / 下单** | P0 | SKU、**可选「点名陪玩」**（客户指定，选人或搜）、备注、提交、校验提示 | **点名** 与 **P0-2 §1.2** 一致；不点名则走公共抢单 |
| S3 | **订单列表** | P0 | 状态标签、空列表、下拉刷新占位（Phase 3 再精修动效） | |
| S4 | **订单详情** | P0 | 状态时间线或简明状态、取消入口（**仅待抢单未接单**，见 P0-1） | 无「联系客服」外链时见 P0-3 §4.2 |
| S5 | **我的** | P1 | 昵称/头像、角色入口（若同端切陪玩）、简单设置项 | 可与 S1 共用导航框架 |

**玩家端 P0 数量**：**4～5**（S1～S4 为核心；S5 可线框）。

---

## 3. 小程序 · 陪玩端

| 序号 | 页面 | 优先级 | 核心内容与状态 | 备注 |
|------|------|--------|----------------|------|
| E1 | **可接单列表（公共大厅 + 点给我的单）** | P0 | 卡片含 **是否点名**；公共单可抢、点名单仅本人可接；抢/接失败 toast | **P0-2 §1.1 / §1.2** |
| E2 | **进行中订单 / 详情** | P0 | 开始服务、完成服务、状态展示 | 与 P2-1 状态名对齐后替换文案 |
| E3 | **陪玩入口 / Tab** | P1 | 与玩家分流、身份切换说明 | 实现方式以 **P1-4** 为准 |

**陪玩端 P0 数量**：**2**（E1、E2）。

---

## 4. 管理后台（PC Web）

| 序号 | 页面 | 优先级 | 核心内容 | 备注 |
|------|------|--------|----------|------|
| A1 | **登录** | P1 | 账号密码或约定方式 | 视觉可极简 |
| A2 | **订单列表** | P0 | 筛选区、表格列、分页占位 | 列字段与接口表一致 |
| A3 | **订单详情** | P0 | 详情字段、操作区：**待抢单态须含「指派陪玩」**（选目标陪玩、确认；与 P0-2/P0-3 一致）、取消、强制状态修正 | **一期必画**指派交互；禁止只保留取消无指派 |
| A4 | **（可选）仪表盘占位** | P2 | 一期可不做 | 防需求蔓延 |

**后台 P0 数量**：**2**（A2、A3）；登录 A1 可组件库。

**信息架构（IA）**：列表 / 详情 / 操作区分区见 **`../../design/dev/D0-3-管理端信息架构.md`**（与 **D0-2 §5** 线框互补）。

## 5. 跨页一致性与组件

| 组件/模式 | 说明 |
|-----------|------|
| **订单状态色标** | 与 P0-1 用户可见状态一致；具体色值设计给定。 |
| **Loading / Toast** | 全小程序统一一种主风格。 |
| **空列表** | 玩家订单空、陪玩大厅空、后台无数据，各一处文案（Phase 1 文案包）。 |
| **错误态** | 网络错误、未登录；避免展示敏感堆栈信息。 |
