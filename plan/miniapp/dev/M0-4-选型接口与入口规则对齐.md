## 5. 玩家 / 陪玩入口与角色可见性规则

### 5.1 真源优先级（争议时按此排序）

| 优先级 | 文档 | 内容 |
|--------|------|------|
| 1 | **`接口表-v0.md`**（登录/用户/角色字段） | 可执行契约 |
| 2 | **`../../product/dev/P1-4-玩家与陪玩入口及Tab分流规则-v0.md`** | Tab 四栏、陪玩二级入口、**一期单角色/测试号**、403 行为 |
| 3 | **`P0-1-一期业务流程说明（demo）.md` §9** | 账号与角色关系的闭合位置说明 |
| 4 | **`P0-5` · E3** | 陪玩入口 / Tab 为 **P1**；**实现方式以 P1-4 为准** |

> **当前状态**：**P1-4 v0.1** 已落库（见上表优先级 2）。**§5.2** 为其执行摘要，与 **P1-4** 冲突时以 **P1-4** 为准。

### 5.2 小程序侧执行规则（与 P1-4 §1～§5 一致）

| 主题 | 规则 |
|------|------|
| **玩家默认路径** | 启动后进入 **底部 Tab**，顺序为 **首页 → 下单 → 订单 → 我的**，与 `pages.json` 一致；对应 **玩家端主路径**（P0-5 S2～S5）。 |
| **陪玩主路径（业务）** | 一期主策略为 **抢单大厅**（**P0-2**）；大厅列表与抢单交互在 **`pages/companion/pending`** 及其后续子页演进（E1）。 |
| **陪玩入口（工程）** | **不设为第 5 个 Tab**。由 **首页内显式入口** `navigateTo` **`pages/companion/pending`**；变更方案见 **P1-4 §3**。 |
| **角色可见性（前端）** | **M1-1 之前**：不承诺按角色隐藏路由（骨架期可任意点进陪玩页）。**M1-1 之后**：以 **后端返回的角色/权限** 为准；非陪玩调用陪玩接口时 **403 + 文案**（错误码见接口表）。**M1-2** 起：按 **P1-4 §5** 做 **无 token / 无权限** 拦截与 toast。 |
| **同 openid 多角色** | **一期不交付**应用内切换；见 **P1-4 §4** 与 **P0-1 §9**。 |

### 5.3 建议产品写入 P1-4 的摘要（便于一次性闭合）

以下可原样迁入产品 **P1-4** 正文，避免双写漂移：

> 一期小程序 **底部 Tab 四栏** 服务 **玩家**：首页、下单、订单、我的。**陪玩抢单大厅** 默认通过 **首页入口** 进入二级页（当前路径 `pages/companion/pending`），不设第五 Tab。用户角色以 **登录接口返回** 为准；非陪玩进入大厅页时列表请求失败展示统一错误提示。若需改为「我的页内角色切换」或「陪玩独立 Tab」，以变更单修订路由与本文。
