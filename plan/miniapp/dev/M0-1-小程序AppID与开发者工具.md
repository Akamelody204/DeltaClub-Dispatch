# M0-1 · 小程序 AppID 与开发者工具

> **对应任务**：小程序负责人工作计划 · Phase 0 · M0-1。  
> **目的**：完成**微信公众平台注册/绑定**、**微信开发者工具**与**仓库访问权限**，使团队成员能在合规前提下打开工程并预览；**仓库内不出现 AppSecret 等密钥**。  
> **依据**：`../../三角洲陪玩派单小程序-快速落实计划.md` Phase 0「小程序」行；`../小程序负责人工作计划.md` §1 总原则；`../../技术栈选型记录.md` 项目范围（仅微信小程序）。

---

## 1. 负责人需完成事项

| 项 | 说明 |
|----|------|
| **注册小程序** | 使用俱乐部/主体资质在微信公众平台注册「小程序」，取得 **AppID**。 |
| **绑定开发者** | 在公众平台「成员管理」中添加开发、体验者账号；与负责人确认角色。 |
| **本地工具** | 安装[微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)，版本与团队统一（建议稳定版）。 |
| **导入工程** | 构建产物目录：`miniapp/dist/build/mp-weixin`（见 M0-2）；在开发者工具中「导入」，**AppID** 选测试号或正式号。 |
| **仓库权限** | 克隆 monorepo 或 `miniapp` 子目录的读/写权限与分支约定（与后端 B0-1 对齐）。 |

---

## 2. `manifest` 与 AppID 写入策略

| 项 | 约定 |
|----|------|
| **`src/manifest.json`** | `mp-weixin.appid` 填入团队**非敏感**的 AppID（AppID 可进库）；**勿**将 AppSecret 写入任何文件。 |
| **多人协作** | 若有人使用个人测试号，可用本地 `manifest` 覆盖层或开发者工具项目设置，**避免**将个人测试号强制提交覆盖他人配置；统一方案由负责人在群公告确认。 |
| **当前仓库状态** | 见本文「完成标志」：未填 AppID 则勾选未完成，直至写入或书面说明「仅工具内绑定」。 |

---

## 3. 安全红线（与总计划一致）

- **禁止**在 `miniapp/` 内提交：`AppSecret`、支付密钥、任意服务端密钥。  
- 若需 CI 上传预览版，使用公众平台**单独密钥**或平台提供的安全变量，**不进 Git**。
